The strategy now has a complete trade lifecycle in code. `paper.fills` carries the full audit (entry + exit rows, shared `a2_intent_uuid`). `paper.positions` reflects only currently-open positions. Re-entry within a single run is allowed. Position-carryover across runs remains out of scope (future Day 29+ work).

## What this does NOT establish

- **Generalization beyond Sep 7 2021.** One episode is not a strategy proof. Sample size = 1 market regime.
- **Real-fee accuracy.** The cost model is conservative. Live fees, real slippage, and venue-specific economics may diverge in either direction.
- **Live execution viability.** PAPER_RESEARCH mode uses synthetic fills with `_NoopFetcher`. Real-venue tests come later in the canary phase.
- **Capacity.** P&L is computed per `quantity_per_intent = 10.0` SOL notional. Scaling considerations (market impact, borrow availability, funding-rate dependencies) are separate questions.

Day 29+ work addresses these gaps en route to canary readiness.
